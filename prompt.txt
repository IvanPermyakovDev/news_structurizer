ЗАДАЧА
Сгенерируй ПЯТЬ (5) синтетических расшифровок новостных выпусков, имитирующих текст после автоматического распознавания речи (ASR) с характерными ошибками. Вывод — строго валидный JSON-массив из пяти объектов. Никаких комментариев, пояснений, Markdown и лишнего текста вокруг — только JSON.

ОБЩИЕ ТРЕБОВАНИЯ К СЮЖЕТАМ
- Каждая новость — про разные события и желательно из разных тематик.
- Избегай клеветы и реально идентифицируемых частных лиц; используй вымышленные имена или широко известные публичные структуры/топонимы без спорных утверждений.
- Даты и цифры могут встречаться, но не обязательны.
- Масштаб события: "local" — локальный, "global" — глобальный.

СТРУКТУРА ВЫХОДА (ОБЪЕКТ НОВОСТИ)
Каждый из 5 объектов массива ДОЛЖЕН иметь поля:
{
  "title":        string — краткий чистый заголовок БЕЗ ошибок,
  "topic":        string — одна из строго фиксированных категорий:
                   ["политика","спорт","экономика","технологии и наука","культура и искусство","экология и климат","mixed"],
  "key_names":    string ИЛИ null — перечисление ключевых имён/организаций через запятую (без ошибок). Если упоминаний нет → null,
  "location":     string ИЛИ null — география события (страна/регион/город) БЕЗ ошибок. Если нет → null,
  "scale":        string — "local" ИЛИ "global",
  "key_events":   string — краткая выжимка сюжета БЕЗ ошибок (1–2 предложения),
  "text":         string — основной «сырой» текст после ASR (с ошибками), ОТ 400 ДО 700 СЛОВ (строго).
}

ТРЕБОВАНИЯ К «ЧИСТЫМ» ПОЛЯМ
- В полях title, topic, key_names, location, scale, key_events — не допускаются орфографические/пунктуационные ошибки, агглютинации, разрывы слов и т.п.
- Язык — русский. Значение topic — строго из списка выше. Масштаб — строго "local" или "global".
- Если нет имён собственных или локации, ставь null.

ТРЕБОВАНИЯ К «ГРЯЗНОМУ» ПОЛЮ text (ASR-имитация)
- Объём: 400–700 слов (НЕ МЕНЬШЕ 400 и НЕ БОЛЕЕ 700). Слова считаются по пробельному разделению.
- Стиль: потоковый эфир, без аккуратных абзацев, допускаются внезапные смены темы/тембра (ведущий ↔ репортёр).
- Распределение ошибок по тексту (ориентировочно суммарно до 100% шума):
  • ~25% — орфографические и фонетические искажения («в маскве», «минестерство», «пратежи»).
  • ~15% — съеденные окончания и агглютинации («президент заяви что», «проводят проверк», «вгороде»).
  • ~10% — слитно/раздельно написанные слова («гос дума», «по данныммчс», «согласноопубликованным»).
  • ~5% — неверная сегментация слов («рос си а не», «транс порт»).
  • ~20% — ошибки пунктуации/границ предложений (почти непрерывная речь, редкие или отсутствующие запятые/точки).
  • ~10% — речевые вставки/сбивки («ээ», «вот», «значит», «как бы», «эээ по данным»).
  • ~10% — повторы/самопоправления («он сказал сказал», «по словам по словам министра»).
  • ~5% — ошибки NER в именах/топонимах (искажения собственных имён).
- Не стремись распределять ошибки равномерно по каждому предложению; естественнее — «кластеры» шумов.
- Допускай переключения регистра, пропуски диакритики и редкие латинские вкрапления (например: «live»), но основа — русский.

ДОПОЛНИТЕЛЬНЫЕ УКАЗАНИЯ К ТЕМАТИКАМ
- Стремись к разнообразию: как минимум 4 разные темы среди 5 новостей; одна новость может быть "mixed".
- Примеры акцентов по темам (необязательно для всех): политика — заявления/реформы; спорт — матч/турнир/трансфер; экономика — рынок/инфляция/сделки; технологии и наука — запуск/исследование/ИИ; культура и искусство — премьера/фестиваль; экология и климат — погода/выбросы/инициативы.

ФОРМАТ ВЫВОДА
- Строго один валидный JSON-массив из 5 объектов, без комментариев и лишнего текста.
- Все ключи и строки — в двойных кавычках, без висящих запятых.
- Пример СТРУКТУРЫ (это только схема, не содержание):
[
  {
    "title": "Краткий заголовок без ошибок",
    "topic": "экономика",
    "key_names": "АО Пример, Мария Петрова",
    "location": "Россия, Москва",
    "scale": "local",
    "key_events": "Краткая выжимка без ошибок.",
    "text": "сырой потоковый текст после asr с ошибками ... (400–700 слов)"
  },
  { ... 4 ещё объекта ... }
]

ПРОВЕРОЧНЫЙ ЧЕК-ЛИСТ ПЕРЕД ВЫВОДОМ
1) Ровно 5 объектов? 2) Валидный JSON-массив? 3) Тема из списка? 4) scale ∈ {local, global}? 5) key_names/location = string или null? 6) Поле text в каждой новости 400–700 слов? 7) Чистые поля — без ошибок, «грязное» поле text — с ASR-ошибками по распределению выше? 8) Нет Markdown и лишних комментариев?
